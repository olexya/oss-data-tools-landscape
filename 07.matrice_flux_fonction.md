# Matrice Flux × Fonction - Classification Bidimensionnelle

> Liste complète navigable/filtrable (stars, dates, licences, top/flop): site généré depuis les catalogues → `docs/` (GitHub Pages). Ce document est une analyse éditoriale.

> **Objectif:** Classifier les solutions selon **deux dimensions**: Typologie de Flux (Streaming/Micro-Batching/Batching) ET Fonction dans l'architecture data.

---

## Introduction

Cette classification bidimensionnelle permet de sélectionner les outils selon:
1. **L'axe temporel**: Streaming, Micro-Batching, Batching
2. **L'axe fonctionnel**: Collecte, Transport, Stockage, Traitement, Analyse, Gouvernance

---

## Les Deux Dimensions

### Dimension 1: Typologie de Flux

| Mode | Latence | Use Case Principal |
|------|---------|-------------------|
| **Streaming** | < 1 sec | Temps réel, alertes, IoT |
| **Micro-Batching** | 1s - 5min | Dashboards quasi temps réel |
| **Batching** | > 5 min | Analytics historique, rapports |

### Dimension 2: Fonction Data

| Fonction | Description | Position Pipeline |
|----------|-------------|-------------------|
| **Collecte** | Capture des données sources | Début |
| **Transport** | Acheminement et routage | Milieu |
| **Stockage** | Persistance et organisation | Base |
| **Traitement** | Transformation et agrégation | Milieu |
| **Analyse** | Visualisation et BI | Fin |
| **Gouvernance** | Orchestration, qualité, sécurité | Transverse |

## Guide de Sélection Bidimensionnel

### Étape 1: Identifier la Fonction

**Question:** Quelle est la position dans le pipeline?

- **Début (Source)** → Collecte
- **Milieu (Routage)** → Transport
- **Base (Données)** → Stockage
- **Milieu (Transform)** → Traitement
- **Fin (Insights)** → Analyse
- **Transverse (Ops)** → Gouvernance

### Étape 2: Identifier le Flux

**Question:** Quelle latence maximale acceptable?

- **< 1 seconde** → Streaming
- **1 sec - 5 min** → Micro-Batching
- **> 5 minutes** → Batching

### Étape 3: Consulter la Matrice

Se référer à la section correspondante (ex: "Collecte × Streaming")

### Exemple de Décision

**Besoin:** Dashboards e-commerce avec métriques temps réel (ventes, stocks)

1. **Fonction:** Analyse (fin de pipeline)
2. **Latence requise:** 10 secondes → Micro-Batching
3. **Matrice Analyse × Micro-Batch:**
 - Options: Superset ●, Grafana ●, Metabase ●
4. **Choix:** Superset (BI riche) + Grafana (monitoring)

## Méthodologie de Classification

### Critères Fonction

Chaque outil est classifié selon sa **fonction primaire**:

1. **Collecte**: Capture initiale depuis sources
2. **Transport**: Acheminement fiable entre systèmes
3. **Stockage**: Persistance et organisation
4. **Traitement**: Transformation et enrichissement
5. **Analyse**: Visualisation et BI
6. **Gouvernance**: Orchestration et qualité

### Critères Flux

Classification selon 3 modes basés sur latence observée en production:

- ● **Primaire**: Usage optimal et recommandé
- ◐ **Secondaire**: Supporté mais non optimal
- ○ **Non supporté**: Non recommandé

### Validation

Chaque classification est basée sur:
- Documentation officielle
- Use cases production
- Benchmarks communautaires
- Architecture patterns établis
